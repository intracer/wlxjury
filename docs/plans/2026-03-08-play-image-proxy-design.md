# Play Image Proxy Design

**Date:** 2026-03-08

## Goal

Serve cached images directly from the Play app without requiring Apache, and track the local cache state with an in-memory registry instead of expensive directory scans.

## Problems Solved

1. **Apache dependency** — currently all `<img>` URLs point at an Apache server configured with `wlxjury.thumbs.host`. Apache serves locally cached files and proxies missing ones to Wikimedia. This requires a separate process and configuration.
2. **Expensive cache scanning** — `LocalImageCacheService.localFileSet()` walks the entire cache directory tree on every bulk operation, which is slow for large contests (38k images × 8 sizes = 304k files).

## Design

### In-Memory Registry

`LocalImageCacheService` gains a `ConcurrentHashMap[String, Unit]` (a concurrent set) keyed by absolute local file path string.

- **Populated at startup**: a single background `Future` calls the existing `localFileSet()` scan once and loads all results into the registry.
- **Updated atomically**: whenever any file is written to disk (by cache-warming downloads or on-demand proxy), its path is added to the registry immediately.
- **Used for all cache checks**: `allSizesCached`, `saveResized`, and the new proxy method all query the registry with O(1) lookups. The `localFileSet()` directory walk is no longer called during request handling.

### ImageProxyController

A new Play controller handles:

```
GET /wikipedia/commons/thumb/*path   controllers.ImageProxyController.serve(path: String)
```

`wlxjury.thumbs.host` is set to the Play app's own host, so all `<img>` URLs generated by `Global.legacyThumbUlr` route through this controller.

### On-Demand Proxy Flow

For each incoming request `/wikipedia/commons/thumb/X/XX/file.jpg/{N}px-file.jpg`:

1. **Registry hit** — serve the local file with `Ok.sendFile`. Done.
2. **Registry miss** — resolve the image record from the database (by filename extracted from the URL path).
   - Parse the filename from the URL; look up `ImageJdbc.findByTitle(filename)`.
   - If not found in DB: proxy the exact URL from Wikimedia transparently (stream to browser, do not cache).
3. **Download source** — use existing `downloadWithRetry` to fetch the source image at the appropriate Wikimedia thumbnail step (same snap-up logic as `downloadAndResize`).
4. **Decode and resize**:
   - Decode the source bytes in memory with `ImageIO.read`.
   - For each target height: **skip if already in registry**, otherwise resize with `scale()`.
   - Serve the requested size bytes to the browser **from memory** (no re-read from disk).
5. **Save to disk in background** — all newly created sizes are written to disk and added to the registry in a background `Future` (not on the response path). The browser receives the response immediately when the resize is complete.

### Changes to Existing Code

| Location | Change |
|---|---|
| `LocalImageCacheService` | Add `registry: ConcurrentHashMap[String, Unit]`; populate at startup; update in `saveResized` and the new proxy method |
| `LocalImageCacheService.saveResized` | Replace `if (!file.exists())` with registry check; add to registry after write |
| `LocalImageCacheService.allSizesCached` (bulk) | Replace `existing: Set[File]` parameter with registry lookup |
| `LocalImageCacheService.runDownloadImpl` | Remove `localFileSet()` call; use registry directly |
| `ImageProxyController` | New controller with `serve(path)` action |
| `conf/routes` | Add proxy route |
| `conf/application.conf` | `wlxjury.thumbs.host` defaults to own host |

### Error Handling

- If the DB lookup fails or returns no result: proxy directly to Wikimedia (best-effort, no caching).
- If the Wikimedia download fails: return 502 Bad Gateway.
- If the local file cannot be written: log a warning, still serve from memory; the next request will retry the download.

### Testing

- Unit tests for the registry: populate from scan, add on write, skip existing.
- Unit tests for the on-demand proxy logic (mock `downloadWithRetry`, mock DB).
- Existing `LocalImageCacheServiceSpec` tests continue to pass (registry replaces `localFileSet()` but behaviour is identical).
- The Docker integration test (`LocalImageCacheServiceDockerSpec`) can be adapted to use the Play proxy route instead of Apache once implemented.
