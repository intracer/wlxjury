# Flyway Schema Baseline Generator

**Date:** 2026-03-25
**Status:** Approved

## Problem

The project has 44 Flyway migrations accumulated over time. This makes it hard to understand the current schema at a glance, and applying all migrations from scratch takes unnecessary time.

## Goal

Consolidate all 44 migrations into a single `B44__Baseline.sql` file that represents the complete current schema. The file is generated by a dedicated test that applies all migrations to a fresh MariaDB container and dumps the resulting DDL.

## Design

### Output file

`conf/db/migration/default/B44__Baseline.sql`

`B44` is the baseline migration convention: it represents the schema at the point after V1–V44 have been applied. The file is only valid when applied to an **empty schema** — it uses `CREATE TABLE` without preceding `DROP TABLE`, so it will fail if any of the tables already exist.

When future migrations V45+ are added, this file stays unchanged — they apply on top of the baseline. If a new baseline is ever needed (e.g., after another 40 migrations), a new `B{N}__Baseline.sql` is generated at that point.

### Test class

`test/db/scalikejdbc/SchemaBaselineSpec.scala`

- Tagged `baseline` on the single example (specs2 mutable style: `"..." in { ... }.tag("baseline")`)
- Excluded from the default `sbt test` run via `build.sbt`
- Does **not** use `testDbApp` from `TestDb` — that method does not expose the container reference, and `TestDb.getApp` is `private`. The test builds its own `MariaDBContainer` and Play app inline using the same config-map pattern.
- Does **not** call `roundDao.usersRef` — that initialization is only needed when ORM round relations are exercised, which this test does not do.

Flow inside the test:
```
container.start()
  → build Play app with container's JDBC URL (inline config map)
  → running(app) { ... }        // Flyway applies all migrations on startup
      → container.container     // unwrap to the underlying Java container
          .execInContainer("mysqldump", ...)
      → write stdout to B44__Baseline.sql
container.stop()
```

### Schema dump

`mysqldump` is included in the official `mariadb:10.6.22` image. The command is passed to `container.container.execInContainer(...)` (the Scala `MariaDBContainer` wrapper does not expose `execInContainer` directly; the underlying Java container is accessed via `.container`):

```
mysqldump
  --no-data
  --skip-lock-tables
  --skip-add-drop-table
  --no-tablespaces
  --ignore-table=wlxjury.flyway_schema_history
  -u WLXJURY_DB_USER
  -pWLXJURY_DB_PASSWORD   ← literal strings, same values as TestDb.UserName/Password
  wlxjury
```

- `--no-data`: DDL only, no row data
- `--skip-lock-tables`: avoids lock issues in the test environment
- `--skip-add-drop-table`: no `DROP TABLE` noise; consequence is the file is only applicable to an empty schema
- `--no-tablespaces`: required because the test container user lacks the `PROCESS` privilege; omitting this causes `mysqldump` to fail
- `--ignore-table=wlxjury.flyway_schema_history`: excludes Flyway's own tracking table. `TestDb.Schema` is `private`, so the schema name `"wlxjury"` is hardcoded here — keep in sync if it ever changes.
- No `--databases` flag: `CREATE DATABASE` / `USE` statements are intentionally omitted; Flyway provides the schema context.

The credentials `-u WLXJURY_DB_USER` and `-pWLXJURY_DB_PASSWORD` are the **literal string values** passed to `MariaDBContainer` (matching `TestDb.UserName` and `TestDb.Password`), not shell environment variable references.

### Opt-in mechanism

Tag the single example in the mutable spec:

```scala
"generate B44 baseline migration" in {
  ...
}.tag("baseline")
```

Exclude from default `sbt test` in `build.sbt` (scoped to `Test`):

```scala
Test / testOptions += Tests.Argument(TestFrameworks.Specs2, "exclude", "baseline")
```

The `exclude` filter applies to all specs2 runs including `testOnly`. To regenerate the baseline, override it explicitly:

```bash
sbt "testOnly db.scalikejdbc.SchemaBaselineSpec -- include baseline"
```

The generated file is committed to git.
